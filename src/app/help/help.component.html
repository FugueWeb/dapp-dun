Glossary

Blockchain: a public, decentralized, permission-less ledger distributed across a peer-to-peer network and validated by a consensus algorithm

Decentralized Application (dApp): an internet-based application that interfaces with a blockchain and is capable of generating valid transactions to make state changes

Smart Contract: a collection of code and data with a specific purpose or capability that resides at an address on a blockchain

Token: a programmable asset or access right managed by a smart contract and accessible only by the individual with the private key for the address holding the token

Hash: a type of digital fingerprint; a hash maps a data input of an arbitrary size through a one-way function into a fixed size string of data, allowing for data integrity and non-repudiation

Decentralized Autonomous Organization (DAO): an organization represented by rules encoded within a smart contract and controlled by shareholders of a digital asset (token) and not influenced by a central authority

ETH vs. DUN vs. Reputation

ETH or ether is the native token of the Ethereum platform and serves as a sort of universal currency for all Ethereum-based applications. It is also used to pay small transaction fees (called gas) whenever you make state changes on the blockchain.

DUN is a token internal to this specific dApp. It is a membership share within the Decentralized United Nations, and thus the more DUN tokens you have the more authority you have because your vote is that much closer to meeting quorum. DUN can also be traded on the token smart contract, provided that the market is open and there is liquidity.

Reputation is an ERC-721 non-fungible token earned for successfully passing a proposal. Within the context of the DAO, it is a cryptographic way of signaling experience and good faith to other participants for past actions.

DUN Membership Onboarding

  1. Ensure Metamask is unlocked and on the Ropsten network
  2. Ensure you have whitelisted the fugueweb.com domain
    Metamask -> Click avatar (top right) -> Settings -> Connections -> Connect
  3. Acquire at least 1 DUN token
  4. `approve` the governance contract 1 DUN token
  5. You are now a member, unless through consensus you are sanctioned or your token has been burned

Example Proposals

Basic proposal (no data): Humanitarian aid package
Note, the amount of ETH must not exceed the balance of ETH held by the governance contract. It is the governance contract, not the sender of the transaction, that will ultimately pay the amount to the recipient if the proposal passes.

amount: 100000000000000000 //.1 ETH (in wei)
recipient: 0x1234...ABCD //the address to receive the funds
description: Aid package to 0x1234
data: 0x //no data

Proposal with data: Burn from an address
When making proposals with the `data` parameter, the `recipient` is always the address of the smart contract where the function will be called. So with `burnFrom` you should include the token contract address.

recipient: 0x4aAB103CC7B0363891Ed4C5059E55bCb9f2Ed14D //token contract address
